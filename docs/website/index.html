Why KIVO

Everything stays on your device.

Most compression tools send your files to a cloud server. KIVO does it all locally — faster, safer, and without any dependency on the internet.

🔒

100% Offline

KIVO makes zero network requests. Your files are processed entirely on-device and never leave your hands. Period.

⚡️

Hardware-Accelerated Video

Uses MediaCodec on Android and VideoToolbox on iOS — the same hardware encoders your OS uses internally. Fast, efficient, and battery-friendly.

🖼️

HEIC & All Modern Formats

Compresses Apple's HEIC/HEIF format natively on iOS and Android, plus JPEG, PNG, WebP, and more — using the OS's own image APIs.

📄

Smart PDF Optimization

Rasterizes images embedded in PDFs at an optimized DPI and JPEG quality. Reduces file size significantly without affecting text or layout.

🌍

Truly Cross-Platform

One codebase, five platforms: Android, iOS, macOS, Windows, and Linux. The same experience everywhere, with platform-native optimizations under the hood.

🆓

Free Forever

No subscriptions, no ads, no usage limits, no watermarks, no account required. KIVO is free software built to help people, not to monetize them.

🛡️

Built for privacy by design

KIVO was created with a single guiding principle: your files are yours. The app requests only the permissions it needs to read the file you select and write the output — nothing more. No analytics SDK, no crash reporter, no ad network. Read the full privacy policy →

The Story

Built out of necessity — and a bit of frustration.

Every time I needed to compress a video or an image before sending it, I had to either use an online tool (and trust it with my files) or deal with complex desktop software. Neither felt right.

So I built KIVO — a tool that does one thing well: compress files locally, fast, without asking for anything in return. No account, no internet, no surprises.

KIVO is built with Flutter, which means a single codebase runs natively on Android, iOS, macOS, Windows, and Linux. Under the hood, it uses platform-native APIs for compression — FFmpeg's hardware encoders for video, the OS image APIs for photos, and a pure-Dart pipeline for PDFs.

Flutter Dart FFmpeg MediaCodec VideoToolbox flutter_image_compress iText / pdf
📱

Mobile — Hardware encoders

Android uses MediaCodec (HEVC + H.264). iOS uses VideoToolbox. No bundled codec libraries — pure OS-native speed.

🖥️

Desktop — Bundled FFmpeg

macOS, Windows, and Linux ship with a bundled FFmpeg binary — zero external dependencies, works offline, works everywhere.

🖼️

Images — Native APIs

flutter_image_compress uses BitmapFactory (Android) and UIImage (iOS) to decode and re-encode, including HEIC/HEIF support.

📄

PDFs — Pure Dart

Each page is rasterized, re-encoded as JPEG at an optimized quality, and rebuilt into a new PDF — all in Dart, no native plugin needed.
